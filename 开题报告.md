## <div align = "center">分布式文件系统</div>

### 1.选题的目的与意义

​	如今随着计算规模和数据量的飞速增长，普通的单机系统已经越来越无法满足用户需求，分布式的系统越来越成为主流。而分布式文件系统就是一个很好的代表，它在多台分布在不同位置的机器上存储文件，并将其组织成一个整体进行管理，用户在使用时就感觉像是在访问单机的本地文件系统一样。这样的设计对于大文件存储，海量小文件的读写，以及并行读写都十分有意义。如今流行的云计算等也与该选题息息相关。一个好的分布式文件系统可扩展性强，容量大，还有较高的容错，现如今还是有较为广泛的应用场景的，有研究价值。而我本人也在这几个学期的学习后对相关的方向十分感兴趣，因此希望能遵循软件开发的流程，完成一个基本的分布式文件系统的软件开发。

### 2.选题思路

​	项目产物的主要功能就是一个在多节点存储文件，有一个主控节点来管控节点信息和负责文件读写的调度，用户拥有对文件的基础的访问、打开、删除、缓存等功能，对于用户的请求，主控节点会在所有的存储节点上找到目标文件所在处并在该节点上完成相应操作，且可以支持多用户同时登陆（因此对文件需要有加锁解锁等操作）。在权限方面做到不同用户对文件有不同访问权限。性能上尽量做到低延迟，高容错。大致开发过程我目前的想法是先完成节点通信的部分，然后再完成主控节点和用户与存储节点的交互部分，在实现了单用户的基础上，在完成对文件的上锁解锁等操作。然后如果真的能把这些部分都完成的话，时间还来得及就在为程序设计一些UI。

### 3.技术路线

​	技术原理：操作系统，分布式系统的相关知识。实践方面还需要python的知识。需要解决的技术难题还是很多的，首先第一点就是节点间通信，分布式文件系统节点间的通信使用的是RPC，但是目前我了解到的python的RPC方式有许多种，比如rpyc和xmlrpc以及Google提供的gRPC，暂时还不知道选哪种，之后会逐一了解。然后节点间的交互，文件的备份与同步等都需要解决，还有上锁等操作，讲实话技术难度还是挺大的。

### 4.相关支持条件

​	采用平台：Visual Studio Code

​	开发环境：Python 3.9.5

​	技术成熟度：目前只有理论知识，实践的打码部分还得慢慢学。
